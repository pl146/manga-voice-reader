Processing Pipeline Stages
Stage 1-2: Capture & Crop
Screenshot: Chrome tabs.captureVisibleTab API, sent as base64
Crop: Extension sends cropRect of the manga image element
Object-fit: CSS object-fit:contain offset correction
Stage 3: AI Bubble Detection
Model: RT-DETR-v2 (168MB ONNX)
Input: 640x640 normalized (ImageNet stats)
Classes: bubble, text_bubble, text_free (SFX)
Orphan rescue: Text outside bubbles grouped via mask
Stage 3b: PaddleOCR Text Detection
Det: PP-OCRv5 mobile (fast, lightweight)
Rec: PP-OCRv5 server (highest accuracy)
Role: Finds text regions + fallback OCR recognition
Stage 4: Text Grouping
Mask: Comic Text Segmenter binary mask
Connected components: 3x1 char kernel separation
Merge caps: 35% page height, 55% width max
SFX filter: Geometry-based pre-filtering
Stage 5: Florence-2 OCR PRIMARY
Florence-2-large-ft: Microsoft VLM, 0.77B params, ~1.5GB VRAM
Task: <OCR> prompt reads bubble crops as whole images
Post-processing: WordNinja word splitting + SymSpell validation
Fallback: PaddleOCR raw text if Florence unavailable
Stage 6: Text Cleanup
Word splitting: WordNinja + SymSpell dictionary
Fragment rejoin: Cross-line OCR fragments reassembled
Corrections: 87 manga-specific word corrections
Formatting: Sentence case, contraction fixes, speech-ready
Stage 7: Kokoro TTS PRIMARY
Kokoro v1.0: 82M params, 54 voices, 24kHz, ONNX
Parallel: All bubbles generated simultaneously (~2s for 6 bubbles)
Pre-buffer: Next audio pre-loaded during playback
Speed: 0.5x - 2.0x adjustable
Stage 8: Playback
Audio: WAV streamed to browser via base64
Controls: Space=pause, Arrows=skip bubble
Auto-read: Triggers on page turn detection
Overlay: Reader, Border, Debug, Hidden modes
